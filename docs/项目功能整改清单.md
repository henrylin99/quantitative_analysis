# 项目功能整改清单

## 1. 文档目标

本文档基于当前 `master` 分支整改后代码状态持续更新，目标不是重复宣传系统能力，而是明确：

- 哪些功能已经部分打通
- 哪些功能仍处于演示、占位或降级状态
- 哪些问题会直接影响对外可用性、可信度和后续二次开发
- 应按什么优先级进行整改

适用代码基线：

- Repository: `quantitative_analysis`
- Branch: `master`
- 最近已完成批次：
  - 投资组合真实列表、详情、创建、删除
  - 优化结果保存为真实持仓
  - 组合内持仓新增、编辑、删除
  - 模型删除前后端入口
  - README / 数据库配置 / 回测与分析 mock 清理

## 2. 总体结论

当前项目应定义为“可二次开发的量化系统原型”，不应定义为“完整交付版系统”。

当前状态可粗分为三类：

- 相对真实可用：历史数据下载脚本、部分数据库模型、部分内置因子计算、基础 Flask/API 骨架、投资组合基础 CRUD
- 部分可用但存在降级：机器学习训练、选股评分、组合优化、回测结果展示、实时分析
- 明显未完成或带演示性质：报告管理、部分实时推送、部分历史文档说明

## 3. 整改优先级定义

- `P0`: 影响真实性、可交付性或安全性，必须优先整改
- `P1`: 核心功能未闭环，影响主要业务使用
- `P2`: 体验、扩展性、维护性问题，可排在核心闭环之后

### 4.3 机器学习模块

#### 4.3.1 训练链路混入模拟目标变量

- 优先级：`P0`
- 问题：
  - 当 `target_type == simulated_return` 时，直接进入模拟训练数据逻辑
  - 当真实未来收益不足时，会生成模拟收益率
  - 当完全无法计算真实收益时，会退回基于特征的模拟目标变量
- 影响：
  - 模型效果失真
  - 容易让使用者误以为训练结果基于真实标签
  - 不适合作为量化研究或实盘依据
- 证据：
  - [app/services/ml_models.py](/Users/henrylin/quant_space/quantitative_analysis/.worktrees/dev-merge/app/services/ml_models.py#L132)

  - [app/services/ml_models.py](/Users/henrylin/quant_space/quantitative_analysis/.worktrees/dev-merge/app/services/ml_models.py#L317)

## 6. 建议整改顺序

### 第一阶段：先消除“假功能”与错误文档

- 修正 README 与启动文档
- 移除或隔离所有自动 mock 展示
- 禁止训练接口在真实模式下使用模拟标签
- 下线未实现入口或加“实验/演示”标签
- 清理硬编码密钥

### 第二阶段：打通核心业务闭环

- 完成真实训练结果链路
- 完成回测真实结果展示
- 完成投资组合基础 CRUD
- 完成最小可用的报告查看链路

### 第三阶段：补工程化能力

- 补测试
- 补任务状态追踪
- 补数据来源审计
- 补异常提示与权限控制

- 配置中不再存在硬编码密钥
- 有最小测试套件覆盖核心链路
