# 整改任务拆分清单

基于 [`项目功能整改清单.md`](/Users/henrylin/quant_space/quantitative_analysis/docs/项目功能整改清单.md) 拆分，目标是便于直接建任务、排期和跟踪。

## P0

### P0-1 修正文档口径与启动说明

- 目标：
  - 统一 README、安装文档、演示文档中的项目定位和启动方式
- 主要问题：
  - “功能完整”与“未完成开发”并存
  - 文档中存在不存在的 `python app.py`
- 交付物：
  - 更新后的 `README.md`
  - 功能状态矩阵
  - 正确启动命令说明
- 验收标准：
  - 新用户按文档可成功启动
  - 文档中不再出现冲突表述

### P0-2 移除回测页面自动 mock 展示

- 目标：
  - 禁止回测失败后自动展示模拟结果
- 主要问题：
  - 当前失败时会 fallback 到 mock 数据
- 交付物：
  - 移除自动 `displayMockResults()` 的逻辑
  - 新的错误提示与空态展示
- 验收标准：
  - 回测失败时页面明确报错，不展示伪造收益曲线

### P0-3 禁止真实训练链路使用模拟标签

- 目标：
  - 让真实训练模式只依赖真实标签
- 主要问题：
  - `simulated_return`
  - 真实 future return 不足时自动生成模拟收益
- 交付物：
  - 训练模式拆分为“真实训练”和“演示训练”
  - 真实训练缺标签时直接报错
- 验收标准：
  - 真实训练接口不再自动降级为模拟目标变量

### P0-4 清理训练接口中的伪造展示字段

- 目标：
  - 移除硬编码训练结果字段
- 主要问题：
  - `model_size: '2.3MB'`
  - 默认兜底成绩易误导
- 交付物：
  - 真实训练结果字段
  - 空值和错误返回规范
- 验收标准：
  - 所有训练结果字段均可追溯到真实训练产物

### P0-5 下线或隔离投资组合页面模拟数据

- 目标：
  - 禁止主路径展示硬编码投资组合
- 主要问题：
  - 页面直接加载模拟组合数据
- 交付物：
  - 真实组合 API 或入口下线方案
- 验收标准：
  - 主导航下不再展示假组合数据

### P0-6 禁用实时分钟数据 legacy mock 实现

- 目标：
  - 防止随机分钟数据进入生产链路
- 主要问题：
  - legacy 路径会生成随机 OHLCV
- 交付物：
  - mock 数据路径封禁
  - 数据源开关与来源标识
- 验收标准：
  - 生产模式下分钟数据只能来自真实数据源

### P0-7 清理硬编码数据库凭据与 Tushare Token

- 目标：
  - 清理代码中的明文凭据
- 主要问题：
  - `app/utils/db_utils.py` 中存在硬编码配置
- 交付物：
  - `.env` 驱动的配置读取
  - 缺失配置校验
- 验收标准：
  - 代码仓库不再包含明文密钥和数据库密码

## P1

### P1-1 完成自定义因子能力验收

- 目标：
  - 验证 `dev` 上已接入的表达式白名单因子计算
- 交付物：
  - 支持表达式列表
  - 正反向测试样例
  - 错误提示规范
- 验收标准：
  - 常见公式可算，异常公式可明确报错

### P1-2 建立内置因子真实性校验样例

- 目标：
  - 对核心内置因子做样例级验证
- 交付物：
  - 至少 5 个因子验证样例
  - 输入字段依赖文档
- 验收标准：
  - 因子值与样例计算结果一致

### P1-3 替换模型训练页面模拟进度

- 目标：
  - 用真实任务状态替换随机进度条
- 交付物：
  - 轮询或 WebSocket 任务状态接口
  - 前端状态机
- 验收标准：
  - 页面状态与后端任务状态一致

### P1-4 下线未实现的评分与优化选项

- 目标：
  - 让 UI/API 只暴露真实可用能力
- 交付物：
  - 方法支持矩阵
  - 隐藏或实验标识机制
- 验收标准：
  - 用户看不到名义支持但实际未完成的方法

### P1-5 打通回测展示字段与后端结果

- 目标：
  - 关键回测指标全部由后端给出
- 交付物：
  - 统一的回测结果 schema
  - 前端图表字段映射调整
- 验收标准：
  - 收益、回撤、风险指标均可追溯到后端结果

### P1-6 完成投资组合最小闭环

- 目标：
  - 实现基础 CRUD 和再平衡
- 交付物：
  - 组合列表
  - 组合详情
  - 创建/编辑/删除
  - 再平衡
- 验收标准：
  - 页面上所有操作按钮均有真实后端逻辑

### P1-7 处理实时推送中的 demo 数据

- 目标：
  - 替换 `demo_portfolio` 和模拟新闻
- 交付物：
  - 真实组合推送来源
  - 真实新闻/公告源或功能下线
- 验收标准：
  - 推送内容不再依赖 demo 默认值

### P1-8 处理报告管理页 TODO 面板

- 目标：
  - 决定隐藏还是补齐报告管理能力
- 交付物：
  - 页面下线方案，或最小功能实现方案
- 验收标准：
  - 页面保留则可用，未完成则不对用户暴露

### P1-9 建立基础测试套件

- 目标：
  - 为核心链路建立最小回归保护
- 交付物：
  - `tests/` 目录
  - API 级测试
  - 服务级单元测试
- 验收标准：
  - 核心流程可本地执行回归

## P2

### P2-1 实现模型删除闭环

- 目标：
  - 清理模型文件与数据库残留
- 验收标准：
  - 删除后状态一致，无孤儿记录

### P2-2 完善异常提示与空态展示

- 目标：
  - 让“无数据”“未配置”“权限不足”“功能未开放”状态可区分
- 验收标准：
  - 不再用统一的笼统报错替代真实状态

### P2-3 建立数据来源审计能力

- 目标：
  - 让主要业务数据可以追溯来源、生成时间和任务批次
- 验收标准：
  - 核心数据表可追踪到采集来源和同步任务

### P2-4 统一功能状态标记机制

- 目标：
  - 对“已实现/实验中/演示模式/未开放”建立统一标识
- 验收标准：
  - 前后端和文档中的功能状态表述一致
