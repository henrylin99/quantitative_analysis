# 多因子模型系统使用指南

## 项目定位

当前仓库应视为用于学习和二次开发的原型系统，不应按“完整交付版量化平台”理解。

- 适合：功能验证、代码研究、二次开发
- 不适合：未经补齐前直接作为生产交付、实盘系统或对外承诺的完整平台
- 当前原则：主路径仅保留真实可用能力，演示/占位/模拟能力持续下线中

## 启动方式

当前仓库统一使用真实入口启动：

```bash
python run_system.py
```

按提示选择初始化数据库或启动 Web 服务。

默认访问地址：

- Web 界面：`http://localhost:5000`
- API 根路径：`http://localhost:5000/api`

## 当前功能状态

### 已实现

- 内置因子基础链路
- 自定义因子表达式白名单校验
- 机器学习模型创建与真实训练任务轮询
- 模型删除前后端入口
- 基础选股评分
- 等权重、均值方差、风险平价、因子中性组合优化
- 基础回测结果返回与前后端字段对齐
- 投资组合真实创建、详情、持仓增删改、组合删除、优化结果落库

### 部分实现

- 实时分析主路径已移除 demo/mock 默认值，但真实新闻源和部分推送能力仍未接入
- 报告分析页已移除演示数据，但管理能力仍较弱
- 投资组合页面尚未接入再平衡入口和更完整的批量运维能力

### 未实现或未开放

- Black-Litterman、行业约束等高级优化能力
- 稳定的报告管理闭环，当前仅保留报告列表与生成功能入口
- 统一的数据来源审计与功能状态标记机制

## 推荐使用路径

1. 运行 `python run_system.py`
2. 初始化数据库
3. 启动 Web 服务
4. 优先使用以下已收口页面：
   - 因子管理
   - 模型管理
   - 股票评分
   - 组合优化
   - 投资组合
   - 回测验证

## 使用说明

### 因子管理

- 可查看内置因子
- 可创建受限表达式的自定义因子
- 可通过接口或页面发起因子计算

### 模型管理

- 可创建模型定义
- 可发起真实训练任务
- 页面训练状态由后端任务轮询驱动
- 可删除模型

### 组合优化与投资组合

- 可执行当前保留的优化方法
- 可将优化结果保存为真实持仓
- 可创建真实组合并维护持仓
- 组合页当前不提供再平衡入口

### 回测验证

- 页面不会在失败后自动展示 mock 结果
- 关键结果字段以真实后端返回为准

## 注意事项

- 若缺少数据库配置或数据源配置，部分功能会直接报错而不是降级为模拟结果
- 实时新闻推送当前未开放，页面展示已按未开放标识处理
- 历史文档、截图和分析材料可能仍有少量过期描述，应以 `README.md` 和当前测试约束为准

## 后续整改重点

- 模型训练链路的更完整状态与异常提示
- 实时推送去 demo 数据
- 报告管理页的下线或最小可用实现
- 投资组合再平衡入口
- 数据来源审计与统一状态标记
